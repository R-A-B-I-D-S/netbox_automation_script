# Автоматизация загрузки оборудования в Netbox 🚀

Скрипт для автоматической генерации интерфейсов коммутаторов и добавления реального сетевого оборудования в базу данных NetBox по инвентарному списку.  

    *Важно:* Этот проект предназначен только для личного использования или обучения. Никогда не публикуйте свои токены API в открытом доступе!
---
## Описание ⚙️

Проект решает задачу быстрого наполнения базы данных сетевого инвентаря реальными устройствами. Скрипт выполняет следующие действия:

    Создаёт сайт (Site), локацию (Location) и стойки (Rack), если они ещё не существуют.
    Загружает устройства (Device) со всеми их атрибутами: производитель, модель, серийный номер, IP-адрес, стойка, позиция и т.д.
    Для каждого устройства автоматически создаёт все физические и логические интерфейсы согласно шаблону модели.
    Назначает устройству Primary IP-адрес.
---

## Установка и настройка 🛠️

Для работы скрипта требуется Python >= 3.9 и библиотека pynetbox.
Шаг 1: Создание виртуального окружения

```bash
python -m venv env
source env/bin/activate # Linux/macOS
env\Scripts\activate.bat # Windows
```
Шаг 2: Установка зависимостей

```bash
pip install pynetbox requests
```
Шаг 3: Подготовка файлов конфигурации

Создайте следующие файлы в корневой директории проекта:
Файл .env

Этот файл хранит секретные данные и пути к файлам с данными. Пример содержимого:
Переменная	Описание
API_KEY	Токен пользователя с правами администратора в NetBox.
URLNB	URL вашего сервера NetBox с добавлением /api. Например: https://your-netbox-instance.example/api.
SWITCHES_FILE	Путь до JSON-файла с инвентарём устройств. См. пример ниже.
INTERFACES_FILE	Путь до папки с JSON-шаблонами интерфейсов для разных моделей. См. пример ниже.
LOGDIR	Директория для сохранения логов. По умолчанию рядом со скриптами: ./logs.

Пример заполнения:
```ini
API_KEY="<ВАШ_ТОКЕН>"
URLNB="https://your-netbox-instance.example/api"
SWITCHES_FILE=switch_inventory.json
INTERFACES_FILE=./interface_templates/
LOGDIR=./logs
```
Файл switch_inventory.json

Хранит список конкретных устройств, которые необходимо добавить в NetBox. Каждый объект описывает одно устройство с полным набором его характеристик.

Папка interface_templates/*

В этой папке хранятся отдельные JSON-файлы для каждой модели коммутатора. Имя файла должно соответствовать названию модели в инвентаре (например, interface_sr-s4650x-48fq.json). Это позволяет динамически загружать порты для любой модели.

Пример файла (interface_s5731-h48t4xc.json)
(На основе предоставленного тобой примера)

```json
{
    "s5731-h48t4xc": [
        {"name": "GigabitEthernet0/0/1", "type": "copper-1ge"},
        {"name": "GigabitEthernet0/0/2", "type": "copper-1ge"},
        {"name": "GigabitEthernet0/0/3", "type": "copper-1ge"},
        ... // Остальные порты
    ]
}
```
## Использование 💡

Запустите основной скрипт:

```bash
python add_devices_to_netbox.py
```
После выполнения вы увидите новые объекты в разделе DCIM → Devices вашей панели NetBox.
Как это работает?

    Скрипт читает переменные окружения из .env.
    Он открывает файл инвентаря (SWITCHES_FILE), указанный в окружении.
    Для каждого устройства в инвентаре он проверяет наличие соответствующего шаблона интерфейсов в папке, указанной в INTERFACES_FILE.
    Если шаблон найден, все интерфейсы из него создаются вместе с устройством при отправке запроса в NetBox.
    Устройство добавляется в NetBox с полным набором атрибутов: производитель, модель, роль, платформа, сайт, локация, стойка, позиция, серийник и т.д.
    Интерфейсу, указанному в поле interface инвентарного файла, назначается IP-адрес из поля ipaddress.
